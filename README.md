# ChurnEngine_marketplace-subscription-retention-engine
An end-to-end SQL framework for identifying behavioral churn signals and calculating unit economics (LTV/CAC) in a two-sided professional services marketplace.

Project Overview: Explain that this project identifies "at-risk" users by monitoring drops in marketplace engagement (Demand) before they officially cancel their subscription.Business Logic:The Problem: High-value customers churn when "Demand" is high but engagement with "Supply" (experts) drops.The Solution: A SQL-based monitoring routine that flags users with a >50% engagement drop.Tech Stack: SSMS (SQL Server), CTEs, Window Functions, and Cross Joins.
